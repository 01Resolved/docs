## GET /api/v1/proposal

Returns a list of proposals for a DAO with full decision market analytics.

**Parameters:**

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `slug` | string | No | Project slug |
| `status` | string | No | Filter: `resolved`, `lived` (live), `initiated` |
| `limit` | number | No | Items per page |
| `page` | number | No | Page number |

```bash
curl -H "x-api-key: YOUR_KEY" \
  "https://api.01resolved.com/api/v1/proposal?slug=metadao&status=resolved&limit=10"
```

<Accordion title="Example Response">
```json
{
  "meta": { "totalItems": 37 },
  "data": [
    {
      "id": 8344,
      "title": "Engage Groom Lake Security Initiative - $360,000 USDC",
      "publicKey": "FG93KeoUdUmBtTcxWhJZicc6ef824qccpFdesbw2RJjM",
      "creationDate": "2026-05-15T03:43:13.000Z",
      "endDate": "2026-05-18T03:43:13.000Z",
      "totalTrade": 263,
      "totalTradeVolume": "31067.19",
      "totalUniqueTraders": 10,
      "result": "approved",
      "status": "resolved",
      "netPassVolumeTilt": "22.71",
      "netPassTraderTilt": "16.67",
      "passAndFailTWAPSpread": "3.11",
      "twapThreshold": "-3.0",
      "twapThresholdMargin": "6.11",
      "twapAndFinalOutcomeSpread": "1.94",
      "organizationSlug": "metadao",
      "tokenSymbol": "META"
    }
  ]
}
```
</Accordion>

| Field | Description |
|-------|-------------|
| `publicKey` | On-chain proposal address. Use as the `{publicKey}` path param in detail endpoints |
| `result` | Final outcome: `approved`, `rejected`, or `""` (pending) |
| `netPassVolumeTilt` | Net % of volume economically aligned with pass. Formula: `((pass-buy vol + fail-sell vol) - (fail-buy vol + pass-sell vol)) / total × 100`. **Fail-sell is Pass-aligned.** Positive = net capital was economically aligned with approval. This is a market-implied signal, not proof that the proposal was objectively value-accretive. |
| `netPassTraderTilt` | Net % of traders economically aligned with pass. Same logic applied to trader counts. Positive = more participating wallets were economically aligned with approval than rejection. This is a market-implied signal, not proof that the proposal was objectively value-accretive. |
| `passAndFailTWAPSpread` | Spread between pass and fail TWAP. Formula: `((pass TWAP - fail TWAP) / fail TWAP) × 100` |
| `twapThreshold` | Minimum pass edge required for approval. A positive threshold means the pass TWAP must exceed the fail TWAP by that amount. A negative threshold means the pass TWAP may underperform the fail TWAP by up to that amount and still pass. |
| `twapThresholdMargin` | How far above the threshold: `passAndFailTWAPSpread - twapThreshold`. Negative = rejected. |
| `twapAndFinalOutcomeSpread` | `((TWAP outcome price - final outcome price) / final outcome price) × 100`. Small = accurate market signal. |

## GET /api/v1/proposal/analytics

Returns aggregate proposal counts for a DAO.

**Parameters:** `slug`

```bash
curl -H "x-api-key: YOUR_KEY" \
  "https://api.01resolved.com/api/v1/proposal/analytics?slug=metadao"
```

**Response:** `{defeated, succeeded, total}`: MetaDAO: 9 defeated, 28 succeeded, 37 total.

## GET /api/v1/proposal/{publicKey}

Returns full detail for a single proposal including the complete proposal text.

```bash
curl -H "x-api-key: YOUR_KEY" \
  "https://api.01resolved.com/api/v1/proposal/FG93KeoUdUmBtTcxWhJZicc6ef824qccpFdesbw2RJjM"
```

<Note>The `content` field contains the full proposal body in markdown format.</Note>

## GET /api/v1/proposal/{publicKey}/pass-edge

Returns the current TWAP status and pass edge. **The most direct way to check whether a live proposal is passing.**

```bash
curl -H "x-api-key: YOUR_KEY" \
  "https://api.01resolved.com/api/v1/proposal/FG93Keo.../pass-edge"
```

<Accordion title="Example Response">
```json
{
  "data": {
    "proposalStatus": "succeeded",
    "threshold": "-3",
    "twapStartTime": "2026-05-16T03:43:13.000Z",
    "passTwapPrice": "2.893608526547",
    "failTwapPrice": "2.806367224118",
    "approveRejectTwapDifference": "6.11",
    "proposalTwapStatus": "approve"
  }
}
```
</Accordion>

| Field | Description |
|-------|-------------|
| `proposalTwapStatus` | Current market signal: `approve`, `reject`, or `pending` (TWAP not yet started) |
| `approveRejectTwapDifference` | Current percentage spread. Positive = pass favored. |

## GET /api/v1/proposal/{publicKey}/market-overview

Returns live market statistics for a proposal.

| Field | Description |
|-------|-------------|
| `passPoolTVL` | Total USD in pass conditional pool. Formula: `USDC + (Pass tokens × spot price)` |
| `failPoolTVL` | Total USD in fail conditional pool. Formula: `USDC + (Fail tokens × spot price)` |

---

## GET /api/v1/proposal/{publicKey}/orders

Returns all individual trades on a proposal's decision market.

**Parameters:** `publicKey` (path), `limit`, `page`

| Field | Description |
|-------|-------------|
| `marketType` | `pass` or `fail` |
| `direction` | `buy` or `sell` |
| `value` | USD value of this trade |
| `price` | Execution price per governance token (USD) |
| `size` | Governance tokens traded |
| `marketDirectionTradeConc` | This trade as % of total market-direction volume |
| `marketDirectionConc` | This trader's total volume in this direction as % of direction's total |

---

## GET /api/v1/proposal/{publicKey}/orders/csv

Downloads all orders as a CSV file.

## GET /api/v1/proposal/{publicKey}/price-chart

Returns 15-minute interval price history for pass and fail markets, including TWAP values.

<Note>`approvedTwap` and `rejectedTwap` are `0` before the TWAP start time.</Note>

**Response fields per data point:** `timestamp`, `approvedPrice`, `rejectedPrice`, `spotPrice`, `approvedTwap`, `rejectedTwap`

---

## GET /api/v1/proposal/{publicKey}/flow-chart

Returns time-series net pass-aligned volume tilt over time.

**Parameters:** `publicKey` (path), `timeInterval` (**required**: `15m`, `1h`, `4h`, `1d`, `max`)

| Field | Description |
|-------|-------------|
| `netPassVolumeTilt` | Net % of volume in pass direction at this interval |
| `cumulativeTilt` | Running total of pass-aligned minus fail-aligned capital (USD) |

---

## GET /api/v1/proposal/{publicKey}/threshold-chart

Returns time-series pass edge relative to the approval threshold. **The clearest chart for whether a proposal is passing.**

**Parameters:** `publicKey` (path), `timeInterval` (**required**: `15m`, `1h`, `4h`, `1d`, `max`)

| Field | Description |
|-------|-------------|
| `threshold` | Required pass edge for approval |
| `passEdge` | Current pass edge at this timestamp. Above threshold = proposal is passing. |

## GET /api/v1/proposal/analytics/{publicKey}/outcome-strength

Returns a detailed breakdown of pass vs fail conviction by trader count and capital.

<Accordion title="Example Response">
```json
{
  "data": {
    "passThresholdClearance": 6.11,
    "traderByEconomicDirection": {
      "passAligned": 58.33,
      "failAligned": 41.67,
      "passBuyTraders": 8,
      "failSellTraders": 6
    },
    "volumeByEconomicDirection": {
      "passAligned": 61.35,
      "failAligned": 38.65,
      "passAlignedVolume": 19060.51,
      "failAlignedVolume": 12006.68
    },
    "lateConvictionLevel": "high"
  }
}
```
</Accordion>

| Field | Description |
|-------|-------------|
| `passThresholdClearance` | `(Pass and Fail TWAP Spread %) - (TWAP Threshold %)`. Positive = passing above threshold. |
| `traderByEconomicDirection.passAligned` | % of traders economically aligned with pass. Pass-aligned = pass-buy + fail-sell traders. **Fail-sell is Pass-aligned.** |
| `lateConvictionLevel` | `\|NVT_twap - NVT_pre_twap\|`. Low < 5%, Medium 5–12%, High ≥ 12%. |

## GET /api/v1/proposal/analytics/{publicKey}/conviction-chart

Returns hourly net pass-aligned capital over time.

| Field | Description |
|-------|-------------|
| `preTwapAccumulation` | Net pass-aligned capital before TWAP window. Formula: `(pass-buy + fail-sell) - (pass-sell + fail-buy)` |
| `twapAccumulation` | Net pass-aligned capital during TWAP window |
| `netPassAlignedCapital` | Running USD net position per hour. Positive = pass pressure dominant. |

---

## GET /api/v1/proposal/analytics/{publicKey}/trade-frequency-chart

Returns the distribution of traders by number of trades placed.

**Response:** `{"1": N, "2": N, "3-5": N, "6-10": N, "11-20": N, "21+": N}`
