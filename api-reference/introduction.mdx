## Base URL

```
https://api.01resolved.com
```

All endpoints are prefixed with `/v1/`.

## Authentication

Include your API key in the `x-api-key` header on every request.

```bash
curl -H "x-api-key: YOUR_API_KEY" \
  "https://api.01resolved.com/v1/dao/treasury/overview?slug=metadao"
```

## How 01Resolved Metrics Should Be Read

01Resolved metrics are designed to make ownership coin activity legible. Treasury value, NAV, mNAV, runway, decision market volume, pass edge, and trader alignment are analytical fields derived from onchain activity and 01Resolved classifications. They are not audited financial statements and should not be read as standalone investment recommendations.

In decision markets, the API distinguishes between outcome, price signal, capital alignment, trader participation, and conviction over time. A proposal passing does not mean it was objectively correct. It means the market cleared the configured threshold under the rules of that decision market.

## Power Endpoints

The most useful endpoints for common use cases:

| Use Case | Endpoint |
|----------|----------|
| Weekly ecosystem snapshot | `GET /global-dashboard/projects-launch` |
| All active proposals right now | `GET /global-dashboard/decision-markets` |
| Is this live proposal passing? | `GET /proposal/{key}/pass-edge` |
| Full proposal text | `GET /proposal/{key}` → `content` field |
| Treasury health for one project | `GET /dao/treasury/overview?slug=` |
| All trades for one DAO ever | `GET /trading-activity/{slug}/trading-activities` |
| Failed ICO history | `GET /global-dashboard/graveyard-launches` |
